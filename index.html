Breve explicación del protocolo HTTP

En el Protocolo de Transferencia de Hipertexto (HTTP), la comunicación entre clientes y servidores se realiza mediante solicitudes y respuestas. Los métodos HTTP, como GET y POST, indican el propósito de la comunicación, mientras que las URL, como https://www.ejemplo.com/pagina, identifican los recursos para encontrar mas información sobre el protocolo esta detallado en el rfc 2616.

Estructura del programa

Este programa Java, compuesto por cuatro clases (Cliente, EncontrarArchivo, Peticion y Servidor), constituye un servidor web básico que maneja solicitudes HTTP GET y envía respuestas correspondientes. A continuación, se resume la funcionalidad de cada clase:

1. Cliente: Inicia el proceso, procesa la solicitud del usuario y envía solicitudes HTTP GET al servidor.

2. EncontrarArchivo: Clase dedicada a la búsqueda y manejo de archivos en el directorio especificado.

3. Peticion: Procesa las solicitudes HTTP del cliente, determinando el tipo y manejando la respuesta correspondiente.

4. Servidor: Escucha continuamente en el puerto especificado, inicia hilos para gestionar múltiples clientes y maneja la configuración y respuestas HTTP.

Este conjunto de clases sigue operando en un bucle infinito, esperando conexiones y garantizando el funcionamiento continuo del servidor web.

Como usar el programa

Cómo Ejecutar el Programa:

1. Descargar un IDE: Descarga un entorno de desarrollo integrado (IDE) como IntelliJ IDEA desde la página oficial.

2. Abrir el IDE: Una vez descargado, abre el programa y carga la carpeta del proyecto con el contenido deseado.

3. Modificar la Clase Servidor: En la clase Servidor, especifica el puerto y el directorio donde se encuentran los archivos que deseas mostrar en tu página web.

4. Ejecutar el Programa: Haz clic en el botón de reproducción (play) para ejecutar el programa.

5. Abrir el Navegador: Abre tu navegador y ingresa tu IP actual junto con el puerto. Si tienes una página llamada index.html, se mostrará automáticamente; de lo contrario, agrega /nombre_archivo.html después del puerto para cargar el contenido deseado.
